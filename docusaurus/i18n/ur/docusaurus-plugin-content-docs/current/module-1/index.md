# ماڈیول 1: روبوٹک اعصابی نظام (ROS 2)

فزیکل AI اور ہیومینوئڈ روبوٹکس: مجسم ذہانت کے ماڈیول 1 میں خوش آمدید۔ اس ماڈیول میں، آپ Robot Operating System (ROS 2) کے بارے میں سیکھیں گے، جو روبوٹک ایپلی کیشنز کے لیے اعصابی نظام کا کام کرتا ہے۔

## سیکھنے کے مقاصد

اس ماڈیول کو مکمل کرنے کے بعد، آپ قابل ہوں گے:
- ROS 2 آرکیٹیکچر کے بنیادی تصورات کو سمجھنا
- مواصلات کے لیے ROS 2 نوڈز بنانا اور منظم کرنا
- انٹر-پروسیس مواصلات کے لیے ٹاپکس اور سروسز کو لاگو کرنا
- rclpy کا استعمال کرتے ہوئے Python ایجنٹس کو جوڑنا
- URDF کا استعمال کرتے ہوئے ہیومینوئڈ روبوٹ ماڈلز بنانا

## جائزہ

Robot Operating System (ROS) روبوٹ سافٹ ویئر لکھنے کے لیے ایک لچکدار فریم ورک ہے۔ یہ ٹولز، لائبریریز، اور کنونشنز کا ایک مجموعہ ہے جو مختلف روبوٹک پلیٹ فارمز میں پیچیدہ اور مضبوط روبوٹ رویے بنانے کے کام کو آسان بنانے کا مقصد رکھتا ہے۔ اس ماڈیول میں، ہم ROS 2 پر توجہ مرکوز کریں گے، جو سیکیورٹی، ریئل-ٹائم کارکردگی، اور ملٹی-روبوٹ سسٹمز کے لیے بہتر خصوصیات کے ساتھ ROS کی اگلی نسل ہے۔

ہیومینوئڈ روبوٹکس کے لیے ROS 2 خاص طور پر اہم ہے کیونکہ یہ مواصلاتی بنیادی ڈھانچہ فراہم کرتا ہے جو روبوٹ کے مختلف اجزاء کو بیک وقت کام کرنے دیتا ہے۔ سینسر ڈیٹا پروسیسنگ سے لے کر actuator کنٹرول تک، ROS 2 روبوٹک سسٹمز کی تقسیم شدہ نوعیت کو ممکن بناتا ہے۔

## پیشگی شرائط

اس ماڈیول کو شروع کرنے سے پہلے، آپ کے پاس ہونا چاہیے:
- Python پروگرامنگ کی بنیادی سمجھ
- کمانڈ-لائن انٹرفیس سے واقفیت
- روبوٹکس تصورات کی بنیادی معلومات (مددگار لیکن ضروری نہیں)

## ماڈیول کی ساخت

یہ ماڈیول کئی اسباق میں تقسیم ہے:
- ROS نوڈز: ROS 2 سسٹمز کے بنیادی بلڈنگ بلاکس
- ٹاپکس اور سروسز: ROS 2 میں مواصلاتی پیٹرنز
- rclpy برج: Python ایجنٹس کو ROS 2 سے جوڑنا
- ہیومینوئڈز کے لیے URDF: ہیومینوئڈ روبوٹس کے لیے روبوٹ ماڈلز بنانا

آئیے ROS نوڈز کو دریافت کرنے سے شروع کریں، جو کسی بھی ROS 2 سسٹم کے بنیادی بلڈنگ بلاکس ہیں۔
